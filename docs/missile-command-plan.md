# Missile Command 実装メモ

## 目的
- Rust 上で `Missile Command` ライクな防衛ゲームを再構築する。
- 第1段階は macOS でのプレイ成立を最優先にする。
- 将来的な Windows / Linux 展開を見据え、描画・入力・ウィンドウ制御は単一クレートに寄せる。

## 採用方針
- ランタイムは `macroquad` を採用する。
- 理由:
  - macOS でフルスクリーン、キーボード、マウス入力、2D 描画が単一依存でまとまる。
  - Windows / Linux にも同じコードで持っていきやすい。
  - 今回のゲームはスプライト資産よりも、線・円・簡易 HUD を高速に描けることが重要。

## 第1段階の実装対象
- フルスクリーン起動
- 照準移動
  - カーソルキー
  - マウス移動
- 発射操作
  - `z`: 左基地
  - `x`: 中央基地
  - `c`: 右基地
- 制御
  - `space` / `p`: 一時停止
  - `q`: 強制終了
- ルール
  - 3基地、6都市、防衛対象の消失
  - 敵ミサイルの波状攻撃
  - 自軍ミサイルの着弾爆風による迎撃
  - ウェーブ進行
  - 残弾ボーナス、都市生存ボーナス
  - スコア閾値による都市復旧

## 現在の追加仕様
- タイトル画面
- ゲームオーバー後のリスタート導線
- 生成音ベースの効果音と簡易 BGM
- 設定画面
  - 基地ごとの装弾数
  - 爆風半径
  - 難易度
- スマートボムの部分実装
  - プレイヤー爆風を検知すると横方向へ回避する
  - 難易度に応じて出現波と出現率を変える

## 初版での設計分割
1. `main.rs`
   - ウィンドウ設定
   - メインループ
2. `game.rs`
   - ゲーム状態
   - 入力処理
   - ウェーブ制御
   - ミサイル、爆風、当たり判定
   - 描画

## ルール上の意図的な簡略化
- まずは通常弾道ミサイルと分裂弾を実装対象にする。
- スマートボム、爆撃機、衛星は次段階の追加要素とする。
- ただし、コード構造は敵種追加を前提とする。

## ウェーブ設計
- 各ウェーブで敵数と速度を少しずつ上げる。
- 中央基地は左右基地より高速に発射する。
- ウェーブ終了時に:
  - 未使用ミサイル数に応じたボーナス
  - 生存都市数に応じたボーナス
  - 次ウェーブ開始時に基地弾数を補給

## 今回の構築順
1. 設計メモを残す
2. `macroquad` 導入
3. 画面・入力・HUD を実装
4. ミサイル、爆風、当たり判定を実装
5. ウェーブ進行、スコア、ゲームオーバーを実装
6. `cargo fmt` / `cargo check` で確認
